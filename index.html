    Все про телефонию¶

    Типы исходящих¶

    1) из софтфона

    2) из диалога

    3) перевод на сотрудника

    4) прозвоны

    Софтфон¶

    Канал исходящего¶

    Когда совершается исходящий звонок из софтфона,то канал по которому он идет определяется в Настройках → Прочие → Системные настройки → Канал для исходящих вызовов. Канал по умолчанию - это канал с тегом outgoing, который где-то но должен быть, если не в этом КЦ, то в другом. Соответственно исходящий из одного КЦ может проходить через канал другого КЦ.

    Номер исходящего¶

    Номер для исходящего может быть разным. Прежде всего он определяется через выбор на софтфоне, над кнопкой вызова.

    Выбор на софтфоне - это список номеров, который принадлежит КЦ, его можно изменить в БД. Тут есть два варианта: выбор номера либо Использовать номер канала.

    Выбор номера, тут все ясно.

    Использовать номер канала¶

    Следующий выбор Использовать номер канала, он задается Настройках → Каналы и операторы → Телефония → канал1 → Редактировать → Исходящий номер. Там также есть выбор номера, и еще Использовать номер контакт-центра.

    Использовать номер контакт-центра¶

    Он задается в Настройках → Прочие → Системные настройки → Номер для исходящих вызовов. Номер по умолчанию - это номер: 73452212222. Это царь номер Этажей.

    Звонок из диалога (перезвон)¶

    Он совершается через кнопку вызова в диалоге

    Канал исходящего¶

    Исходящий из диалога идет по каналу диалога.

    Номер исходящего¶

    Номер канала или номер КЦ если: 1) звонок будет проходить не через канал телефонии 2) звонок идет через канал телефонии, а в его настройках стоит Использовать номер контакт - центра.

    Звонок специалисту и прозвоны в РИЕС¶

    Звонок специалисту¶

    Речь прежде всего про звонок специалисту из софтфона во время диалога, т.е. перевод звонка на специалиста. Потому что если делать просто исходящий из софтфона, то правила поведения точно такие же как и у обычного звонка, единственное отличие - это вторая категория для сотрудника (первая - это категория исходящего).

    Канал исходящего¶

    Исходящий идет по каналу диалога.

    Номер исходящего¶

    Выбор идет аналогично исходящего из софтфона: номер из софтфона → номер канала → номер КЦ → дефолтный номер

    Прозвоны Риес¶

    Канал исходящего¶

    Канал может быть как в самом КЦ, так и в другом.

    Номер исходящего¶

    Номер будет меняться в зависимости от КЦ. При звонке специалистом, находящимся в КЦ1, звонящим через канал КЦ2, номер исходящего будет от КЦ1 (выбранным в настройках). Номер определяеся геолокацией (КЦ) оператора.

    Иерархия назначения номера для исходящих вызовов.¶

    Старая иерархия:

    • Выбранный из списка номер.
    • Номер канала (если это перезвон - номер входящего канала, если просто исходящий - номер канала исходящих вызовов данного КЦ).
    • Номер КЦ канала.

    Новая иерархия:

    • Выбранный из списка номер.
    • Номер КЦ оператора.

    Таким образом из иерархии исключается влияние номера КЦ канала.

    Софтфон¶

    Привязка номера к оператору¶

    Или же номер оператора зарегистрирован в системе. Это когда введен номер телефона (софтфон, поле ввода под статусом), который принадлежит оператору.

    Особенности работы номера у оператора и сотрудника¶

    Например, номер зарегистрирован в системе, т.е. за ним стоит оператор_2. Если ты в разговоре с клиентом переведешь звонок на этот номер, да ещё и не слепым переводом, а нормальным, то звонок переведется на оператора_2.

    Если номер не зарегистрирован в системе, и при этом прописан в users у какого-то пользователя/специалиста, то при переводе разговора с клиентом это будет обычный перевод специалисту.

    Если предположить, что такое возможно: короткий номер не на операторе и не в users, то это клиент.

    Формат номеров операторов¶

    Номер оператора может иметь следующий вид: 1XXX, 5XXX, 0YYXXXX

    Расшифровка 0YYXXXX

    0YY - код региона (ГИБДД)
    XXXX - порядковый номер
    Условно в Тюмени номер оператора может быть 0721234

    Максимальное количество людей, которые могут быть выведены если у них один и тот же подменный номер - это пять. Также выводится статус риелтора. Если он уволен, то появляется подсказка и символ:

    Статус риелтора (символ возле ФИО)¶

    Статус риелтора хранится в таблице users (РИЕС) - поле work_status

    • onvacation
    • maternity
    • sick
    • no_working_temp
    • no_tickets
    • agent_practice
    • test Примечание:
    • cтатус "В работе" не выводится
    • если user_status = deleted, то work_status уже не важен
    • user_status может быть только deleted или active

    Звонки: сотрудник, клиент, оператор¶

    В системе звонки курируют между тремя категориями лиц: клиенты, операторы, специалисты (сотрудники).

    Специалист или сотрудник¶

    Специалист не может быть оператором. Он может никогда в жизни не узнать, что в Этажах есть такая система Омни. Кто может быть специалистом? Например, риелтор в Урюпинске, ипотечник, юрист, программист, иначе говоря, это сотрудник Этажей.

    У него номер телефона может быть коротким номером (как у оператора), длинным (обычный номер), может быть и тот, и тот.

    Оператор¶

    Оператор отличается от специалиста тем, что номер оператора зарегистрирован в системе. Т.е. оператором может быть только человек в омни (с правами на омни), в специалистом может быть кто угодно, чей номер есть в таблице риеса в users.

    Номер телефона: только короткий номер, подробнее см. Формат номеров операторов.

    Примечание: См. Номер оператора зарегистрирован в системе.

    Номер исходящего (над кнопкой Вызов)¶

    Выбор селекта приводит к сохранению выбранного номера для исходящих звонков у одного человека.

    Местоположение настройки¶

    Выбранный номер (точнее идентификатор из таблицы omni_caller_ids) сохраняется в настройки аккаунта (accounts_account_settings). Имя настройки - omni_selected_callerid

    Особенности настройки¶

    • Сохранить можно только один номер

    Возможный вопрос: У пользователя, если он сменит КЦ, он же может сохранить номер1 в КЦ1, номер2 в КЦ2, и при смене КЦ быть с выбранным номером?

    Ответ: В текущей реализации не может. Если он сменит КЦ и выберет новый номер - настройка перезапишется.

    Слепой перевод¶

    Есть звонок у оператора О1, он хочет соединить с оператором О2, то он набирает номер О2 в софтфоне, нажимает на Перевести звонок и может завершить звонок разговор до того, как О2 возьмет трубку - это называется слепым переводом.

    При слепом переводе диалог не переходит к О2, а остается у О1, будто это перевод на специалиста.

    Если при входящем при слепом переводе клиент сбрасывает, приходит второй лишний hangup вместе с record.

    Сброс оператора, сброс программно¶

    Сброс оператора¶

    О1 сбрасывает О2

    О1 в софтфоне нажимает на красную кнопку, завершая разговор с О2

    Сброс программно¶

    О1 на панели телефонии нажимает зеленую кнопку, завершая разговор с клиентом

    Желтушки¶

    Звонок завершен¶

    Появляется * после окончания разговора клиентом * когда исходящий звонок оператора клиент сбросил * когда оператор начал звонить, но сбросил аппаратно трубку

    Вы положили трубку¶

    Только когда оператор завершил звонок программо

    Странное поведение¶

    Возможные ситуации, проблемы¶

    Телефонный звонок не отображается¶

    Если есть диалог по каналу телефонии (в разговоре или просто неактивный) или идет разговор в диалоге чата/мессенджера, то при поступлении нового звонка, оператор его не видит (во вкладке Новые не отображается) и кнопка Принять/Отклонить не появляется.

    Телефонный звонок может появиться с другим телефонным звонком (вкладка Отложено)¶

    Если отложить текущий телефонный диалог на n минут, то новый звонок появляется в системе (принцип: один звонок В работе сохраняется), но, спустя n минут, даже при активном звонке, возвращается первый телефонный диалог из Отложены. Таким образом, может быть несколько телефонных диалогов В работе, но разговор может быть только у одного.

    Пропадают кнопки Автоподхвата/Отклонения, исчезает диалог¶

    1) Идет звонок

    ФР: Диалог появляется в новых после второго обновления страницы , когда есть звонок

    2) Обновить страницу дважды

    ФР: Кнопки автоподхвата/отклонения пропадают и звонок тоже, а со стороны клиента говорят "все операторы заняты" - и это все на протяжении 40 сек, а потом звонок, кнопки все возвращается в нормальное положение. В диалоге при этом появляется сообщение "Звонок отклонён автоматически" (см. ПС).

    Это не странность. Это защита от зависания операторского интерфейса. При включенном автоподхвате после назначения звонка на кого-то, бэк-энд даёт некоторое время, чтобы фронт отправил команду на подхват звонка. Если этого не происходит, срабатывает автоотклонение звонка для этого оператора, чтобы диалог мог перейти следующему. Так как у тебя в системе скорее всего больше нет свободных операторов - клиенту проигрывается "все операторы заняты"
